## Core Web Vitals Improvement Checklist

### Largest Contentful Paint (LCP)
- [x] Hero images use `fetchpriority="high"`
- [x] Font loading uses `font-display: swap` with preload
- [x] Critical CSS is loaded early (inline or preload)
- [x] Code splitting implemented for route-based lazy loading
- [x] Images above the fold are not lazy-loaded
- [x] Preconnect to font origins (`fonts.googleapis.com`, `fonts.gstatic.com`)

### First Input Delay (FID) / Interaction to Next Paint (INP)
- [x] JavaScript is code-split and lazy-loaded per route
- [x] Event handlers are lightweight and non-blocking
- [x] Third-party scripts (GTM, GA4) loaded asynchronously
- [x] Web Workers used for heavy computations where applicable
- [x] React lazy/Suspense used for page components

### Cumulative Layout Shift (CLS)
- [x] Images have width/height attributes or aspect ratio containers
- [x] Fonts use `font-display: swap` to prevent FOIT
- [x] Ads and embeds have reserved space
- [x] Dynamic content (banners, modals) doesn't shift page content
- [x] Skeleton loading screens maintain layout during data fetch

### Additional Performance
- [x] DNS prefetch for external domains (GTM, Google Analytics)
- [x] Resource hints (preconnect, prefetch) configured
- [x] Gzip/Brotli compression enabled at server level
- [x] Cache-Control headers set for static assets
- [x] Service worker configured for offline support (PWA)
- [x] Image optimization with lazy loading for below-fold content
- [x] `decoding="async"` added to non-critical images
