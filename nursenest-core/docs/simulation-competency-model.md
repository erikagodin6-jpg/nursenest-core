# Simulation Competency Model

Date: 2026-06-01
Status: Institutional simulation competency model

Source of truth: `docs/simulation-gap-analysis.md`.

No institutional dashboard, public route, learner-facing page, or production profile system is modified by this document.

## Competency Domains

| Domain | Simulation Evidence Required |
| --- | --- |
| Clinical Judgment | Recognize cues, analyze cues, prioritize hypotheses, generate solutions, take action, evaluate outcomes |
| Escalation | Correct threshold, correct communication, correct urgency, correct documentation |
| Patient Safety | Harm prevention, monitoring, scope-appropriate intervention, reassessment |
| Medication Safety | Verification, monitoring, adverse effect recognition, hold/escalation parameters |
| Communication | SBAR, patient/family communication, team updates, handoff |
| Documentation | Objective notes, event documentation, medication documentation, escalation record |
| Emergency Response | Rapid response, code readiness, deterioration, role clarity, post-event evaluation |

## Competency Profile Update Rule

Every completed simulation should update:

- Domain score
- Missed cue profile
- Delayed escalation profile
- Unsafe intervention profile
- Documentation gap profile
- Recommended remediation
- Reattempt interval
- Confidence delta

## Institutional Reporting Outputs

| Output | Purpose |
| --- | --- |
| Individual readiness profile | Shows learner strengths and weaknesses |
| Cohort readiness dashboard | Shows program-level competency gaps |
| Simulation completion report | Confirms exposure to required cases |
| Remediation queue | Assigns lessons, flashcards, questions, and repeat simulations |
| Competency evidence export | Supports school or employer review |
