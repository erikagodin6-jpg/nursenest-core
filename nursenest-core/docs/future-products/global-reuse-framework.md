# Global Reuse Framework

Status: Governance only. Reuse does not authorize publication.

## Asset Classes

Every future asset must be classified as:

- Global
- Country-specific
- Exam-specific
- Profession-specific

## Inheritance Model

```txt
Global Core
→ Profession Supplement
→ Country Supplement
→ Exam Supplement
→ Language Supplement
→ Product Experience
```

## Required Metadata

- Asset ID
- Asset class
- Source asset
- Reuse percentage
- Translation eligibility
- Localization requirements
- Supplement requirements
- Country applicability
- Exam applicability
- Profession applicability
- Review status
- Publication status

## Reuse Rules

- Reuse global clinical science whenever clinically appropriate.
- Create country supplements only for regulation, documentation, terminology, scope, medication governance, or clinical guideline differences.
- Create exam supplements only for blueprint or item-format differences.
- Create profession supplements only for scope and competency differences.
- Do not duplicate assets when more than 80% of content is shared.

## Reuse Percentage Targets

| Asset Type | Target Reuse |
|---|---:|
| Lessons | 85%+ |
| Flashcards | 90%+ |
| Simulations | 95%+ |
| Clinical cases | 90%+ |
| Labs | 95%+ |
| ECG | 95%+ |
| Pharmacology principles | 90%+ |

## Translation Eligibility

An asset may enter translation only when:

- Clinical core is approved.
- Profession supplement is approved.
- Country supplement is approved, if required.
- Exam supplement is approved, if required.
- No duplicate source asset exists.
- Localization notes are complete.

## Goal

Maximize reuse while preserving exam validity, clinical safety, regulatory accuracy, and learner relevance.
