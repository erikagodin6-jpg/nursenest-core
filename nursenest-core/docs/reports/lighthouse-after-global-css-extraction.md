## Compiled CSS Bundle Analysis

The root CSS chunk went from **1.4 MB → 1.17 MB** (−230 KB, −16%).

### Why Reduction is Smaller than Source Reduction

The compiled CSS includes Tailwind v4 utility output (~900 KB), which is generated by scanning
all source files regardless of which CSS file imports Tailwind. Moving custom CSS to child layouts
reduces the *custom CSS* in the root bundle but does not reduce the Tailwind utility output.

**Tailwind utility CSS estimate:** ~900 KB of the 1.17 MB root bundle
**Custom CSS in root bundle:** ~270 KB (down from ~500 KB)

## Why 75+ Is Not Yet Reached

### Root Bottleneck: Tailwind Utility CSS (~900 KB)

The dominant CSS cost is Tailwind v4 generating utility CSS for all classes used across the
entire application (marketing + learner + admin). This ~900 KB utility output ships to every
route in the root CSS chunk because `@import "tailwindcss"` is in the root-layout globals.css.

**This cannot be reduced by CSS file splitting alone.**

### Path to 75+ Mobile Score

| Change | Est. Gain | Effort |
|---|---|---|
| 1. Tailwind CSS-per-route splitting (custom plugin or purgecss per bundle) | +15–20 pts | Very High |
| 2. Move `@import "tailwindcss"` to per-layout CSS entries (Next.js CSS layering) | +10–15 pts | High |
| 3. Convert learner-only Tailwind utilities to `@apply`-free CSS (reduce scan surface) | +5–10 pts | Medium |
| 4. ISR/CDN caching for homepage (remove force-dynamic for default state) | +5–8 pts | Medium |
| 5. Lazy-hydrate below-fold sections (IntersectionObserver) | +3–5 pts | Medium |
| 6. Remove framer-motion from desktop initial bundle | +2–3 pts | Low |
